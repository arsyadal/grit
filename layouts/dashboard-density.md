# Dashboard Density Module

Use this prompt layer when generating dashboards, admin panels, analytics tools, or data-heavy interfaces that must balance information richness with clarity.

## Intent
Control information density so the interface feels capable and serious without becoming cluttered or visually noisy.

## Prompt Template
```text
Apply this dashboard density module.

Dashboard rules:
- Prioritize information hierarchy before visual styling
- Group related data clearly
- Use density strategically: rich enough to feel powerful, clean enough to remain scannable
- Do not oversimplify the interface into oversized cards with too little substance
- Do not overcrowd the screen with equally loud modules
- Use framing, spacing, and scale to separate primary metrics, secondary panels, and utilities
- Navigation, filters, tables, charts, and summary cards must feel like one system
- Mobile adaptation should preserve usefulness, not just collapse everything into long stacks

Deliver:
- Dashboard or admin UI code
- Short explanation of density strategy and hierarchy logic
```

## Output Criteria
- The UI should feel operational and credible
- High-value information should be obvious immediately
- Dense areas should remain readable and navigable

## Notes
If the model makes the dashboard too empty, add:

```text
Increase information richness and operational depth while preserving clear hierarchy and scanability.
```
