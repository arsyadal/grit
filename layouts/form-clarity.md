# Form Clarity Module

Use this prompt layer when generating forms, onboarding flows, settings panels, or any interface where user input clarity matters more than decorative styling.

## Intent
Make form-heavy interfaces feel clear, trustworthy, and easy to complete without becoming bland or default-looking.

## Prompt Template
```text
Apply this form clarity module.

Form rules:
- Prioritize comprehension and completion speed
- Labels, helper text, field grouping, and error states must be obvious
- Inputs, selects, toggles, and buttons must feel like one coherent system
- Use spacing and framing to reduce cognitive load
- Separate primary actions, secondary actions, and destructive actions clearly
- Avoid cluttered settings walls and avoid oversized empty form layouts
- Multi-step or grouped forms should reveal structure immediately
- Mobile adaptation must preserve clarity, tap comfort, and reading order

Deliver:
- Form or settings UI code
- Short explanation of grouping, hierarchy, and interaction clarity
```

## Output Criteria
- The form should feel easy to understand at a glance
- Important fields and actions should stand out without visual noise
- Error, focus, and helper states should support the user clearly

## Notes
If the form feels too generic, add:

```text
Increase clarity through stronger grouping, more deliberate field hierarchy, and sharper action framing without adding unnecessary decoration.
```
